Eu não recomendo ferramentas porque estão em alta. A maioria das listas de "melhores plugins de 2026" já está velha quando você lê, e todas citam as mesmas coisas. Esta é uma lista diferente. São as cinco que eu abro todo dia, e cada uma é um princípio que este blog defendeu, agora entregue como software que você pode instalar.

Esse é o filtro inteiro. Não popular. Convicção instalável.

**[Superpowers](https://github.com/obra/superpowers)**, de Jesse Vincent. Um framework de skills agênticas que impõe um fluxo de verdade: spec, plano, teste primeiro, depuração sistemática. Vem como plugin do Claude Code e funciona com outros agentes também. É o [loop guiado por spec](/pt-br/posts/2026-05-19-the-spec-driven-loop/) como software instalável. Eu uso porque ele segura a disciplina nos dias em que eu pularia, e são esses os dias que precisam dela.

**[claude-mem](https://github.com/thedotmack/claude-mem)**, de Alex Newman. Um plugin de memória persistente. Ele captura o que o agente fez numa sessão, comprime, guarda localmente, e devolve as partes relevantes para a sessão seguinte. O contexto sobrevive entre sessões e entre compactações. É o [conserte o sistema, não a saída](/pt-br/posts/2026-05-31-fix-the-system-not-the-output/) automatizado. A decisão fica lembrada sem eu reexplicar amanhã.

**[Obsidian](https://obsidian.md)**. Um app de notas em Markdown local, onde um vault é uma pasta simples de arquivos que um agente lê diretamente, sem exportar. É o [seu contexto de IA pertence ao time](/pt-br/posts/2026-05-22-your-ai-context-belongs-to-the-team/), a base de conhecimento durável embaixo de tudo. Nota honesta: não há servidor MCP oficial, as integrações são feitas pela comunidade. O formato de pasta simples é o motivo de funcionar mesmo assim. O modelo lê os mesmos arquivos que você.

**[Playwright MCP](https://github.com/microsoft/playwright-mcp)**, da Microsoft. O servidor MCP oficial que deixa um agente dirigir um navegador de verdade por snapshots estruturados de acessibilidade, em vez de capturas de tela. É a metade gravar-e-refinar do [confie no pipeline](/pt-br/posts/2026-05-23-trust-the-pipeline/). O agente trabalha a UI real, e você fica com os testes de ponta a ponta confiáveis que saem disso.

**[ui-ux-pro-max](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill)**. Uma skill de design de UI/UX: estilos, paletas de cores, pares de fontes, orientação de layout e interação que o modelo usa quando constrói interfaces. É o gosto que um modelo cru não tem, a lacuna que o [não apenas use o modelo](/pt-br/posts/2026-05-30-dont-just-use-the-model/) nomeou. É opinativa, e esse é o ponto. Padrões batem uma página em branco que se enche de slop genérico de IA.

## O que isto não é

- Não é um guia de compras. São as que eu uso, não um ranking do campo. Deixei boas ferramentas de fora porque não as uso todo dia, e uma recomendação que eu não posso sustentar não vale nada.
- Não é permanente. Esta lista é verdadeira hoje. Ferramentas nesta área se renovam rápido, e o princípio sobrevive a qualquer uma. Se uma melhor sair no mês que vem, eu troco e o princípio não se mexe.
- Não é a sua stack. A sua deveria combinar com o seu trabalho, não com o meu. Pegue o princípio primeiro, escolha a ferramenta que encaixa depois.

## Fechamento

Olhe os cinco de novo. Cada um é um post que escrevi antes de a ferramenta entrar no meu radar. O método, a memória, a base de conhecimento, os testes, o senso de design. Defendi a ideia, depois achei o software que a tornou real, nessa ordem. Nunca o contrário.

É o único jeito de escolher ferramentas que duram. A ideia é o que você guarda. A ferramenta é só onde ela mora este ano. Pegue o princípio, depois vá às compras.
